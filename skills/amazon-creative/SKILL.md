# Amazon Creative Workflow

Amazon.co.jp商品ページのコンテンツ計画からワイヤーフレーム制作までを実行する。

## ワークフロー

```
Phase 1: コンテンツ計画（何を・どの枠で・どの順で伝えるか）
  → 訴求マップ + リスティングテキスト + 画像訴求テキスト + 動画戦略
  → ユーザー承認
Phase 2: ワイヤーフレーム（HTML/CSS → PDF、テキスト配置・サイズ・フォント含む完成形）
  → デザイン上テキスト調整が必要なら Phase 1 に戻って修正
  → ユーザー承認
  → 素材（写真）はユーザーが準備。レタッチは amazon-image-retouch スキルを使用
```

各フェーズは独立実行可能。「訴求マップだけ」「ワイヤーフレームから」など部分リクエストにも対応する。

## Phase 1: コンテンツ計画

商品コンテキスト（`.claude/amazon-product-context.md`）があれば先に読み込む。なければ必要情報をヒアリング。

### Phase 1 の出力物

**1a. 訴求マップ** — ページ全体の設計図

| 枠 | メディア | 訴求タイプ | 訴求内容（要約） | 必要素材 |
|----|----------|----------|----------------|---------|
| サブ1 | 静止画 | メイン | 白背景商品写真 | 商品写真 |
| サブ2 | 静止画 | 権威 | 保育士推薦 | 保育士写真 |
| ... | ... | ... | ... | ... |
| 動画 | 動画 | 使用シーン | 食事中の使用風景 | 動画撮影 |
| A+1 | A+画像 | 詳細解説 | 素材の特徴 | 商品アップ写真 |

**1b. リスティングテキスト**
- タイトル（全角65文字以内、先頭30文字でスマホ対応）
- 箇条書き5本（各100〜200文字、【見出し】付き）
- バックエンドキーワード（タイトル・箇条書きと重複しない語）
- 商品説明文（A+がある場合は省略可）

**1c. 画像訴求テキスト（各画像のテキスト要素）**

各サブ画像・A+画像について、テキスト要素を列挙する（この段階では配置・サイズ不問）:

```
■ サブ画像2枚目: 権威付け
見出し: 保育士の92%が推薦
本文: 全国保育士協会アンケート（2025年・n=500）
キャプション: ※自社調べ
写真素材:
  調達方法: 撮影
  被写体: 保育士が商品を手に持っている
  構図: アイレベル・上半身
  雰囲気: 明るい・信頼感
  背景: 保育園の室内（ぼかし）
```

**写真素材プロンプトの必須項目:**
- **調達方法**: `AI生成` / `撮影` / `既存素材加工` のいずれか
- **被写体**: 何が写っているか
- **構図**: アングル・フレーミング（俯瞰/アイレベル/クローズアップ）
- **雰囲気**: 明るさ・トーン（ナチュラル/スタジオ/暖色系）
- **背景**: 背景の種類と詳細

**1d. 動画戦略**
- 動画が必要か（動作を伴う機能、Before/After、使用シーンの場合に有効）
- 内容: 何を伝えるか
- 構成: 尺の目安（30〜90秒）、字幕必須、最初の3秒で掴む

## Phase 2: ワイヤーフレーム

### Phase 2 開始前の確認事項

ワイヤーフレーム作成を開始する前に、**必ずユーザーに以下を確認する**:

- サブ画像のキャンバスサイズ（推奨: 縦長 1600×2000px）
- メイン画像のキャンバスサイズ（推奨: 4:5 = 2000×2500px）
- A+は970×600pxで固定（Amazon仕様）

Phase 1で承認された内容をもとに、各画像の構図をHTML/CSSで作成する。

**ワイヤーフレームの目的:** テキストと写真の配置・サイズ・レイヤー関係のみを確定する。色・装飾・仕上げは後工程（Affinity Designer等）。

**ワイヤーフレームで確定すべきこと:**
- 写真エリアの配置・サイズ比率（キャンバスの60%以上を写真が占めること）
- テキストの配置位置・フォントサイズ・ウェイト（スマホで読めるサイズ）
- テキストと写真のレイヤー関係（重なり・前後関係）
- テキスト内のキーワード強調（サイズ変化・太字）

**ワイヤーフレームに含めないこと:**
- 色（白黒グレーのみ）
- 装飾（角丸、影、グラデーション、ボーダー）
- カード型コンテナ（テキストと写真を分断するボックス）

**出力形式**: HTML/CSSファイル。ブラウザで直接プレビュー可能。PNGレンダリングは任意（`scripts/render_html.py`）。

### 構図の基本原則

1. **写真が主役**: キャンバスの60-90%を写真が占める。テキストは写真の上または隣に配置。
2. **レイヤード構図**: `position: absolute` で要素を重ねる。Flexbox/Gridのボックス分割ではなく、グラフィックデザインの構図で考える。
3. **端まで使い切る**: 大きなパディングは不要。写真はキャンバス端まで配置。
4. **スマホで読めるフォント**: 2000pxキャンバスはスマホで約1/4に縮小表示される。最小フォントサイズ36px（スマホで9px相当）。

## 画像サイズ基準

| タイプ | サイズ | 用途 |
|--------|--------|------|
| メイン画像 | 2000×2500px (4:5) | メイン商品画像（検索結果で大きく表示される） |
| サブ画像 | 1600×2000px (4:5) | サブ画像デフォルト（縦長で商品ページ表示を最大化） |
| A+標準 | 970×600px | A+ 標準画像＋テキスト |
| A+比較表 | 各150×300px | A+ 比較表の商品画像 |

※サブ画像のサイズはPhase 2開始前にユーザーに確認する。上記は推奨値。

## Amazon規約: 禁止・リスク表現

| NG表現 | 理由 |
|--------|------|
| 最高、最強、No.1 | 主観的・証明不可 |
| 激安、最安値 | 価格変動するため |
| 送料無料 | Amazon配送条件次第 |
| 他社ブランド名 | 商標権侵害 |
| 治る、痩せる | 薬機法・景品表示法違反 |
| 永久保証 | 消費者契約法上の問題 |
| 価格表示（¥xxx） | 価格変動時に画像更新が必要 |
| レビュー評価（★x.x） | レビュー蓄積で画像更新が必要 |
| 安い・お得・コスパ最強 | 主観的＋価格変動リスク |
