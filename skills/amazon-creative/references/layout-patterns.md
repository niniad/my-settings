# 構図パターン辞書

## 基本原則

**写真が主役、テキストは上に乗る。**

ベンチマーク画像の共通点:
- 写真がキャンバスの60-90%を占める
- テキストは写真の上に重ねるか、写真と同じ視覚平面に配置
- テキストと写真が別ボックスに分断されない
- キャンバス端まで使い切る（大きなパディング不要）
- キーワードが巨大（140px+）で、スマホでも一瞬で読める

**構図 = Webレイアウトではない。**
- Flexbox/Gridで「カード」を並べるのはWebページの発想
- 商品画像は「グラフィックデザイン」の構図で考える
- `position: absolute` で要素を自由に配置し、重なりを作る

## 構図パターン

### photo_side_text（写真＋横テキスト）

写真が55-65%、テキストが残り。テキストは写真に寄せて配置。

```
┌──────────────┬─────────┐
│              │ 見出し    │
│              │          │
│   写真        │ ポイント1 │
│  (55-65%)    │ ポイント2 │
│              │ ポイント3 │
│              │          │
│              │ バッジ    │
└──────────────┴─────────┘
```

- 写真は左端から配置（パディングなし）
- テキストは写真の右端に寄せる（大きな隙間を作らない）
- 適する訴求: メインUSP、こだわりポイント

### full_bleed_overlay（全面写真＋テキスト重ね）

写真が全面。テキストは写真の上に直接配置。

```
┌─────────────────────┐
│ [見出しテキスト]       │
│                     │
│      写真(全面)       │
│                     │
│              バッジ1  │
│    バッジ2            │
└─────────────────────┘
```

- 写真はキャンバス全面（2000x2000）
- テキストは上部/下部/角に配置
- 適する訴求: ライフスタイル、使用シーン

### comparison（比較: 問題→解決）

左右で比較。各セルにテキスト＋写真を配置（カード枠なし）。

```
┌──────────┬──────────┐
│  [見出し全幅]          │
├──────────┬──────────┤
│ ✕ 問題1  │ ◎ 解決1  │
│ (説明)   │ (説明)   │
├──────────┼──────────┤
│ ✕ 問題2  │ ◎ 解決2  │
├──────────┼──────────┤
│ ✕ 問題3  │ ◎ 解決3  │
├──────────┴──────────┤
│ [写真エリア]           │
└─────────────────────┘
```

- 問題側は薄いグレー背景、解決側は白背景で区別
- ボーダーやカード枠は使わない（背景色の違いだけで区別）
- フォントは大きく（問題/解決のタイトルは .sub 52px以上）
- 適する訴求: 従来品との比較、Before/After

### numbered_panels（ナンバリングパネル）

大きな番号＋テキスト＋右側に写真。

```
┌─────────────────────┐
│ [大見出し]            │
├─────────────────────┤
│ 01  問題タイトル [写真]│
├─────────────────────┤
│ 02  問題タイトル [写真]│
├─────────────────────┤
│ 03  問題タイトル [写真]│
├─────────────────────┤
│ [まとめテキスト]       │
└─────────────────────┘
```

- 番号は超大サイズ（120-160px）、薄いグレーで背景的に配置
- 問題タイトルは .heading (80px) で太く
- 各パネルの右端に小さな写真プレースホルダー
- パネル間は細い線で区切る（カード枠ではない）
- 適する訴求: お悩み一覧、機能ポイント一覧

### callout_diagram（商品写真＋吹き出し）

中央に大きな商品写真、周囲にバッジ/ラベルを配置。

```
┌─────────────────────┐
│ [見出し]              │
│  ┌───────────────┐   │
│  │               │ 吹 │
│吹│    商品写真     │ き │
│き│   (大きく)     │ 出 │
│出│               │ し │
│し│               │   │
│  └───────────────┘   │
├───────┬───────┬─────┤
│ 特徴1  │ 特徴2  │特徴3│
└───────┴───────┴─────┘
```

- 商品写真は大きく（幅の60-70%、高さの40-50%）
- 吹き出し/バッジは `position:absolute` で写真の上や周囲に配置
- 下部に特徴カラム（写真＋テキスト）
- 適する訴求: 機能解説、スペック訴求

### spec_diagram（商品詳細図解）

表裏の製品写真に寸法線を重ね、下部に部位アップ。

```
┌─────────────────────┐
│ [商品詳細]            │
├──────────┬──────────┤
│  表面     │  裏面     │
│ (寸法線   │ (寸法線   │
│  重ね)    │  重ね)    │
├──┬──┬──┬──┬──┬──┤
│P1│P2│P3│P4│P5│P6│
└──┴──┴──┴──┴──┴──┘
│ タグ1 タグ2 タグ3 ... │
└─────────────────────┘
```

- 表裏写真は大きく（各45%幅、高さ55-60%）
- 寸法ラベルは `position:absolute` で写真上に配置
- 下部の部位アップは小さなグリッド
- 適する訴求: 寸法、素材、各部位の特徴

### gallery（バリエーション一覧）

グリッドで多数の写真を並べる。

```
┌─────────────────────┐
│ [大見出し]            │
├──┬──┬──┬──┬──┬──┬──┤
│  │  │  │  │  │  │  │
├──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │  │
├──┴──┴──┴──┴──┴──┴──┤
│ [フッター: CTA＋価格]  │
└─────────────────────┘
```

- 見出しは巨大（120px+）
- グリッドはキャンバス幅いっぱいに配置
- フッターにまとめ買い訴求＋価格
- 適する訴求: カラバリ、セット内容

## フォントサイズ基準

2000pxキャンバスはスマホで約500px幅に縮小表示される（約1/4）。

| 役割 | フォントサイズ | スマホ換算 | ルール |
|------|-------------|-----------|--------|
| キーワード | 120-160px | 30-40px | 1画像に1-2語。最も伝えたい言葉。 |
| 見出し | 72-100px | 18-25px | セクションの主題。見出しだけで内容が伝わること。 |
| サブテキスト | 48-60px | 12-15px | ポイント名、短い説明。 |
| 本文 | 40-48px | 10-12px | 詳細説明。読まなくても画像は理解できること。 |
| 注釈 | 36px | 9px | 最小サイズ。これ以下は使わない。 |

**見出しとキーワードだけで内容が伝わること。** 本文以下はスマホでは読みにくいため、補助情報に留める。
