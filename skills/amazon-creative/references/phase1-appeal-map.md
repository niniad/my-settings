# Phase 1: 全体訴求マップ設計

## 目的

商品のすべての訴求ポイントを洗い出し、サブ画像・動画・A+コンテンツの全体で「何を・どの順番で・どのメディアで」伝えるかを設計する。

## 入力情報の収集

以下の情報をユーザーから収集する。不足分は質問して埋める。

### 必須情報

- **商品名・カテゴリ**: 何を売るか
- **ターゲット層**: 誰に売るか（年齢、性別、ライフステージ、悩み）
- **主要な差別化ポイント**: 競合と何が違うか（最低3つ）
- **価格帯**: 高価格帯/中価格帯/低価格帯（訴求トーンに影響）

### あると望ましい情報

- **競合ASIN**: BSR上位の競合商品（サブ画像を分析するため）
- **既存レビュー**: 高評価・低評価の主な内容
- **アンケート結果**: ターゲット層の悩み調査（クラウドワークス等）
- **権威付け素材**: 専門家推薦、受賞歴、特許、検査結果
- **商品写真の在庫**: 手元にある写真の種類と品質

## 訴求ポイントの洗い出し

以下のフレームワークで訴求ポイントを網羅的にリストアップする。

### PASフレームワーク（Problem → Agitation → Solution）

各ターゲットの悩みについて:
1. **Problem**: ターゲットが抱える具体的な問題
2. **Agitation**: その問題が放置されるとどうなるか（感情を揺さぶる）
3. **Solution**: 自社商品がどう解決するか

### 訴求カテゴリの分類

| カテゴリ | 説明 | 例（お食事エプロン） |
|----------|------|---------------------|
| 機能訴求 | 商品の具体的な機能・性能 | 3Dポケット、撥水コーティング |
| 感情訴求 | ターゲットの感情に訴える | 「つけたい！」と言ってくれる |
| 権威訴求 | 第三者からの信頼性 | 現役保育士推薦 |
| 比較訴求 | 競合との差を明示 | マジックテープ vs スナップボタン |
| スペック訴求 | 数値・寸法・素材 | 首回り29-33cm、ポリウレタン |
| 社会証明 | レビュー・販売実績 | ★4.5、累計1万個販売 |

## サブ画像とA+の役割分担

### サブ画像の役割

スクロール中に手を止めさせ、「もっと知りたい」と思わせる。短時間で直感的に伝わる内容を優先。

- 感情に訴える内容（お悩み共感、権威付け）
- 一目で伝わる差別化ポイント
- 購買を後押しする決定打

### A+コンテンツの役割

サブ画像で興味を持ったユーザーが「もう少し詳しく知りたい」時に読む。

- 詳細なスペック・素材説明
- 使い方の手順
- よくある質問への回答
- ブランドストーリー
- 競合との詳細比較表

### 動画の役割

静止画では伝えきれない「動き」と「リアル感」。

- 商品の使用シーン全体
- 動作を伴う機能の実演
- 開封〜セットアップ手順
- サイズ感の実物確認

## 訴求マップの作成手順

1. 洗い出した訴求ポイントを優先度順に並べる
2. 各訴求に最適なメディアを割り当てる
3. サブ画像枠（最大8枚＋動画1-2本）に高優先度の訴求を配置
4. 残りをA+モジュールに配置
5. 全体を通して読んだときのストーリーの流れを確認
6. 重複がないか確認（サブ画像とA+で同じことを言っていないか）
