# Phase 2: レイアウト・モジュール設計

## 目的

Phase 1の訴求マップをもとに、各画像のワイヤーフレーム（テキスト配置、写真配置、グリッド構成）を設計する。A+コンテンツはモジュール選択も行う。

## 画像サイズの決定手順

### 1. カテゴリ調査

出品カテゴリのBSR上位10商品のサブ画像を確認し、以下を調べる:

- 正方形（1:1）vs 縦長（1:1.25〜1:1.6）の採用比率
- 1画像あたりの情報量（テキスト量、写真枚数）
- スマホでの見え方（商品画像ギャラリーでの表示サイズ）

### 2. サイズ選定基準

| 状況 | 推奨サイズ | 理由 |
|------|-----------|------|
| 大半の競合が正方形 | 2000×2000px | ギャラリー表示で最大サイズ |
| 情報量多い系（サプリ等） | 1600×2000px | タップ最大化で読ませる前提 |
| 迷ったとき | 2000×2000px | 汎用性が高い |

### 3. A+モジュール別の画像サイズ

A+モジュールは種類ごとに画像サイズが決まっている。代表的なもの:

| モジュール名 | 画像サイズ（推奨） |
|------------|------------------|
| スタンダード画像とテキスト | 970×600px |
| スタンダード4つの画像とテキスト | 各220×220px |
| スタンダード比較表 | 各150×300px |
| スタンダードテキスト付き画像(左) | 300×300px |
| スタンダードテキスト付き画像(右) | 300×300px |
| 画像とテキストのオーバーレイ | 970×600px |

※Amazonの仕様変更により変わる可能性があるため、制作時にSeller Centralの最新仕様を確認すること。

## レイアウトパターン辞書

以下のパターンから、訴求内容に合ったものを選択する。

### pattern: hero_text_side（ヒーロー写真＋横テキスト）

```
┌─────────────────────┐
│ [大見出し]            │
├──────────┬──────────┤
│          │ ポイント1  │
│  大きな   │ ポイント2  │
│  写真     │ ポイント3  │
│          │           │
└──────────┴──────────┘
```

**適する訴求**: 着け心地のこだわり、機能の詳細解説
**例**: お食事エプロン4枚目（着け心地へのこだわり）

### pattern: grid_comparison（グリッド比較 ✕ vs ◎）

```
┌─────────────────────┐
│ [大見出し]            │
├──────────┬──────────┤
│ ✕ 問題1  │ ◎ 解決1  │
├──────────┼──────────┤
│ ✕ 問題2  │ ◎ 解決2  │
├──────────┼──────────┤
│ ✕ 問題3  │ ◎ 解決3  │
└──────────┴──────────┘
```

**適する訴求**: 競合の問題 vs 自社の解決策
**例**: お食事エプロン5枚目（食べこぼしキャッチ）、6枚目（お手入れ楽々）

### pattern: numbered_list（ナンバリングリスト）

```
┌─────────────────────┐
│ [大見出し]            │
├─────┬───────────────┤
│ 01  │ 問題1 + 詳細   │
├─────┼───────────────┤
│ 02  │ 問題2 + 詳細   │
├─────┼───────────────┤
│ 03  │ 問題3 + 詳細   │
└─────┴───────────────┘
│ [まとめテキスト]       │
└─────────────────────┘
```

**適する訴求**: お悩みポイント一覧、機能ポイント一覧
**例**: お食事エプロン3枚目（お悩みポイント3選）

### pattern: endorsement（推薦者・権威）

```
┌─────────────────────┐
│ [大見出し]            │
├──────────┬──────────┤
│  人物     │ 肩書き    │
│  写真     │ 名前      │
│          │ コメント   │
├──────────┴──────────┤
│ ポイント1 | ポイント2 | ポイント3 │
└─────────────────────┘
```

**適する訴求**: 専門家推薦、インフルエンサー推薦
**例**: お食事エプロン2枚目（現役保育士さんも推奨）

### pattern: spec_diagram（商品詳細図解）

```
┌─────────────────────┐
│ [商品詳細]            │
├──────────┬──────────┤
│  表面     │  裏面     │
│ (寸法線)  │ (寸法線)  │
├──┬──┬──┬──┬──┬──┤
│P1│P2│P3│P4│P5│P6│  ← 部位別アップ写真
└──┴──┴──┴──┴──┴──┘
```

**適する訴求**: 寸法、素材、各部位の特徴
**例**: お食事エプロン7枚目、マザーズリュック商品詳細

### pattern: before_after（Before/After or 問題→提案）

```
┌─────────────────────┐
│ [大見出し]            │
├──────────┬──────────┤
│ 日々の    │ 当社からの │
│ お手入れが│ ご提案     │
│ 大変…    │           │
│ [写真]   →  [写真]    │
├──────────┴──────────┤
│ 比較ポイント × 3       │
└─────────────────────┘
```

**適する訴求**: 従来品の問題→自社商品の解決
**例**: お食事エプロン5枚目、6枚目の上部構成

### pattern: feature_callout（機能コールアウト）

```
┌─────────────────────┐
│ [大見出し]            │
├─────────────────────┤
│ [写真] ← テキスト1    │
│ [写真] ← テキスト2    │
│ [写真] ← テキスト3    │
│ [写真] ← テキスト4    │
│ [写真] ← テキスト5    │
└─────────────────────┘
```

**適する訴求**: 複数の機能・特徴を羅列
**例**: マザーズリュックPOINT 03（その他のこだわりポイント）

### pattern: variation_gallery（バリエーション一覧）

```
┌─────────────────────┐
│ [大見出し]            │
├──────────┬──────────┤
│ [写真A]  │ [写真B]   │
│ カラーA   │ カラーB    │
├──────────┼──────────┤
│ [写真C]  │ [写真D]   │
│ カラーC   │ カラーD    │
└──────────┴──────────┘
```

**適する訴求**: カラバリ、セット内容一覧

## テキスト原稿の生成

各画像のテキストを以下のレイヤーで設計する:

### テキストヒエラルキー

| レイヤー | 役割 | フォントサイズ目安 | 例 |
|---------|------|-----------------|-----|
| 大見出し | 画像全体のテーマ | 48-64px | 「食べこぼしをしっかりキャッチ」 |
| 中見出し | セクションタイトル | 32-40px | 「3Dポケット」「体にフィット」 |
| 本文 | 説明テキスト | 20-28px | 「ひっくり返して使う」 |
| キャプション | 補足・注釈 | 16-20px | 「※使用環境により異なります」 |
| 手書き風 | 感情・共感テキスト | 可変 | 「毎日のことだから…」 |

大見出しと中見出しで内容が伝わること。本文以下は補足。スマホのギャラリー表示では大見出しと中見出ししか読めないと想定する。

## ワイヤーフレームの出力形式

各画像について以下を出力する:

```markdown
### 画像3: お悩みポイント3選

**レイアウトパターン**: numbered_list
**サイズ**: 2000×2000px
**背景色**: #FAF7EB

**構成要素**:
- 上部: 大見出し「100人のパパママに聞いたお悩みポイント3選」
- 左上小テキスト: 手書き風「お食事エプロンってどうやって選べばいいの？」
- グリッド3行:
  - 01: 「子供が付けるのを嫌がる!!」+ 泣く赤ちゃん写真 + 詳細2行
  - 02: 「結局食べこぼしてしまう」+ 食べこぼし写真 + 詳細2行
  - 03: 「お手入れが面倒」+ 洗濯写真 + 詳細2行
- 下部: まとめ「同じ悩みを持つ子育て中のスタッフが…」

**必要素材**:
- 泣いている赤ちゃん写真（AI生成 or ストック）
- 食べこぼしシーン写真
- 洗濯シーン写真
```
